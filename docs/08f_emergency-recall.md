# Emergency Recall Notifications
**Objective:** Manage product recalls and revoke related credentials.

## Actors
- Issuing Authority
- Stakeholders

## Initiate Recall
Issuing authority initiates recall using the web application and updates credential status.

## Update Credential Status
Credentials marked as revoked in AATP API.

## Notify Stakeholders
All stakeholders are notified via DIDComm. Define clear messaging protocols for efficient recall communication.

## Acknowledge Recall
Stakeholders confirm receipt and update their systems.

## Ensure Revocation
AATP API confirms revocation status, ensuring all credentials are invalidated.

## Monitor Compliance
Issuing authority monitors compliance with recall procedures, using DIDComm for secure and efficient updates.
